Deploy Raspberry

Posted on 2020-01-19 Edited on 2020-01-21 In Toys Views:

It is not difficult to deploy Raspberry since there are many guides online. This post is to remind myself of some points during the deployment.

  • Use the SD Formatter to format the SD card unless the blank space remains in Linux would lead Raspberry cannot read the SD card.
  • Download the system deploy image (intact) from official site.
  • Use Win32DiskImager to write image into the SD card.
  • Ensure the power supply is safe and stable. If the voltage is not stable, it may even burn the chip. If the voltage is not high enough, the chip cannot work.
  • Insert the SD card into PC again, the console would warn you that one of the partitions in the card requires to be formatted. IGNORE THE WARNING, unless you want to rewrite the system in SD card. If you format the partition, Raspberry cannot read the SD card correctly.
  • (For users deploy raspberry without screen) Enable the SSH. Enter BOOT patition in SD card, create a text named as ‘ssh’, and erase the extension of the txt file or use sudo raspi-config in command.
  • Connect power and reticle to the chip.
  • Use PUTTY to establish the remote desktop connection. Check Raspberry’s IP address on router managing site. If the connection is refused, check the ssh file is created and whether the partition is formatted correctly.
  • User: pi; Password: raspberry
  • Windows + R, input ‘mstsc’ to activate the remote desktop manager. Input the chip’s IP address. After connecting, user, password is above.
  • Then Raspberry is deployed.
