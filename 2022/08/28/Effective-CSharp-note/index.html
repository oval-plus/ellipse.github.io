Effective C#笔记

  1. 用static constructor or static initializer来初始化static property. 如果要用static constructor，单例模式
  2. 优化构造函数
    1. 在构造函数里使用default parameter
    2. 为了new()，准备一个myclass()构造函数方法
    3. myclass().age(12).name();
    4. 顺序
      1. Static variable storage is set to 0.
      2. Static variable initializers execute.
      3. Static constructors for the base class execute.
      4. The static constructor executes.
      5. Instance variable storage is set to 0.
      6. Instance variable initializers execute.
      7. The appropriate base class instance constructor executes.
      8. The instance constructor executes.
  3. 对有继承IDisposable的类使用using来Dispose resource management
  4. 少使用没有用的变量，减轻garbage collector的负担。
    1. 例子一：少在经常使用的method中new一模一样的变量（可以采用readonly来固定住new过一次的结果，或者是采用单例模式的思想）
    2. 例子二：对string少用+=方法（使用format或者是stringbuilder）
  5. 正确写Dispose方法与Finalizer来管理resource，finalizer用来管理unmanaged resource
    1. 防止client未及时调用Dispose()或finalizer引发内存泄漏
    2. 减轻Garbage Collector负担
    3. 基类继承IDisposable接口时，子类也需要写dispose，并且调用基类的dispose方法来确保所有的内存资源都得到妥善管理
    4. 如果没有unmanaged resource，不要写finalizer
  6. Reference Type V.S. Value Type
    1. Is this type’s principal responsibility data storage?
    2. Is its public interface defined entirely by properties that access its data members?
    3. Am I confident that this type will never have subclasses?
    4. Am I confident that this type will never be treated polymorphically?
  7. Enum指定0
  8. Prefer immutable Atomic variable to deal with the multithread and validation. Easier to maintain and to code
  9. limit visiblity of the type to reduce the maintaining pressure
  10. define and implement interface rather than inheritance:
    1. interface: behaves like; inheritance: is a
    2. interface: fewer box and unbox
  11. Delegate to improve the callback
  12. Event
  13. reference type的read-only处理方法：value type, immutable type, 用接口暴露功能性方法而不暴露本身，wrapper clone一个
  14. serializable type
  15. 给API定合理的api
  16. covariance & contravariance
  17. 对继承类型定义新行为时偏向override重写，而不是新增eventhandler
  18. IComparable and IComparer. IComparable defines the natural order for your types.A type implements IComparer to describe alternative orderings
  19. avoid icloneable: 1. easier to implement and use 2. 如果一个类实现了icloneable, 那么它的成员变量和继承类型全部需要实现icloneable
  20. 慎用new修饰符
  21. 不要overload基类中已经存在的方法，因为这不方便维护，而且还有强转基类之类的问题。
    35 and 36 学会使用PLINQ
  22. 善用AggregateException in Parallel Algo.
    38 to 44. Dynamic and Expression
    • Expression: lazy implementation
    • dynamic: runtime中才能知道是什么类，Generic: compiler阶段就已经知道是什么了
  23. Box and unbox minimization
  24. Exception class合理定义：只有对client来说需要执行不同操作的时候再定义一个新的exception
  25. Strong Exception Guarantee
  26. unsafe code: /unsafe C# compiler exception or pinvoke or COM interfaces
    49 and 50. Make sense Assembly:
    • To create a CLS-compliant assembly, you must create an assembly whose public interface is limited to those features in the CLS specification
    • assembly之中的内容尽可能合理，解耦
