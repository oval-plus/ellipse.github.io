Cannot start MySQL solution

Posted on 2019-11-29 Edited on 2020-01-21 In Study Views:

After installing MySQL 8.0, some problems emerged, like MySql cannot be started in cmd.

  • First, check whether /bin is already added into the path. If not, add the /mysql/mysql server/bin to the path.

  • Second, check whether the path in cmd is correct. It must direct to the path of where mysql server is installed, such as /mysql/mysql server/bin.

    • If two steps above are done but that still not work, try to use mysql -u root -p to force start or mysqld --console to check the error message.

  • Third, mysqld --initialize is required before input net start mysql. If net start mysql does not work after initialization, delete the data folder in /mysql/mysql server folder, and initialize again.

  • If cmd reminds that ‘MySQL cannot start’ but it does not report any error, use mysqld --console to check the error. If the problem is caused by the port is used, like 3306, use netstat -ano|findstr 3306 to find the pid of process listening 3306 port, then kill it with taskkill /f /pid *, replacing * with the pid.

    The problem would be solved. net start mysql would work and enjoy coding.
    DO NOT FORGET TYPE SEMICOLON AT THE END OF CHANGING PASSWORD COMMAND ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '******';, involving number, symbols, capital and lower case letters.
